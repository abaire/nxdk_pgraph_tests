32// Indicates that all values being pushed in a pb_push* invocation should be sent to the same command.
33//
34// The default behavior is for pgraph commands that do not operate on fixed size arrays to treat multiple parameters as
35// invocations of subsequent pgraph commands.
36// E.g., NV097_INLINE_ARRAY (0x00001818) expects a single parameter which is added to the inline array being built up
37// in graphics memory. Rather than invoke pb_push1 many times over and waste time/space re-sending the
38// NV097_INLINE_ARRAY command, it is desirable to send multiple components (e.g., all 3 coordinates for a vertex) in a
39// single push. Setting bit 30 (i.e., NV2A_SUPPRESS_COMMAND_INCREMENT(NV097_INLINE_ARRAY)) instructs the pushbuffer to
40// treat all the parameters in the command packet as part of the NV097_INLINE_ARRAY command.
41// Failure to set bit 30 would utilize the default behavior, so the first parameter would go to NV097_INLINE_ARRAY, but
42// the next would go to NV097_SET_EYE_VECTOR (0x0000181C), which itself will expect 3 parameters and will at best lead
43// to unexpected behavior and potentially an exception being raised by the hardware depending on how many parameters in
44// total are being sent.
45#define NV2A_SUPPRESS_COMMAND_INCREMENT(cmd) (0x40000000 | (cmd))
